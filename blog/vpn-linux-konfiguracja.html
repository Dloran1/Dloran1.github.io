VPN na Linuxie — konfiguracja krok po kroku

Linux daje kilka wygodnych sposobów na połączenie z VPN: WireGuard (najszybszy), OpenVPN (bardzo kompatybilny) oraz integrację przez NetworkManager. Poniżej zebralem szybkie recepty dla popularnych dystrybucji.

1) WireGuard — terminal (wszystkie dystrybucje)

  1. Zainstaluj pakiety:
    • Ubuntu/Debian: sudo apt update && sudo apt install wireguard resolvconf
    • Fedora: sudo dnf install wireguard-tools
    • Arch: sudo pacman -S wireguard-tools
  2. Skopiuj plik konfiguracyjny xxx.conf do /etc/wireguard/: sudo cp ~/Pobrane/serwer.conf /etc/wireguard/wg0.conf
  3. Uruchom tunel: sudo wg-quick up wg0 (wyłącz: sudo wg-quick down wg0).
  4. Autostart po reboocie: sudo systemctl enable wg-quick@wg0

2) OpenVPN — terminal

  1. Instalacja:
    • Ubuntu/Debian: sudo apt install openvpn
    • Fedora: sudo dnf install openvpn
    • Arch: sudo pacman -S openvpn
  2. Połącz z profilem: sudo openvpn --config ~/Pobrane/profil.ovpn
  3. Odłącz: Ctrl+C w tym samym oknie terminala.

3) NetworkManager — GUI (WireGuard / OpenVPN)

  1. Zainstaluj wtyczki:
    • Ubuntu/Debian (GNOME/KDE): sudo apt install network-manager-openvpn network-manager-wireguard
    • Fedora: sudo dnf install NetworkManager-openvpn NetworkManager-wireguard
  2. Otwórz Ustawienia sieci → + → Import z pliku → wskaż .conf (WG) lub .ovpn (OVPN).
  3. Zapisz i przełącz suwak, aby połączyć.

Kill Switch i ochrona DNS

  • WireGuard (wg-quick): w profilu można mieć reguły PreUp/PostUp blokujące ruch poza tunel (iptables/nft). Prostsza alternatywa: reguły zapory, które dopuszczają tylko interfejs wg0.
  • OpenVPN: użyj firewall-a (ufw/nftables) by zezwolić na ruch wyłącznie przez interfejs tunelu tun0.
  • DNS leak: upewnij się, że w profilu są ustawione serwery DNS dostawcy VPN; na systemach z systemd-resolved sprawdź, czy po połączeniu DNS wskazuje na interfejs VPN.

Szybkie komendy zapory (przykład UFW)

# pozwól tylko przez interfejs wg0
sudo ufw default deny outgoing
sudo ufw default deny incoming
sudo ufw allow out on wg0 to any
sudo ufw enable

Uwaga: to twardy Kill Switch — zanim włączysz, upewnij się, że masz poprawny profil i wyjście awaryjne (np. dostęp lokalny).

Diagnostyka i testy

  • Adres IP: po połączeniu powinien wskazywać kraj serwera VPN.
  • DNS: sprawdź, czy odpowiadają serwery DNS dostawcy (zobacz VPN a wycieki DNS).
  • Routing: ip route powinno mieć domyślną trasę przez interfejs tunelu.
  • Szybkość/ping: przejdź do checklisty testów.
